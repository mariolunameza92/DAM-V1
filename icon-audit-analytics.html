Guía de anotaciones
A1KPI icon box: HTML usa msi xs (18px) pero .an-kpi-icon .msi { font-size:16px } en CSS lo sobreescribe. La clase HTML es letra muerta.
A2Card title: HTML usa msi xs (18px) pero .an-card-title .msi { font-size:15px } lo reduce. El CSS manda; la clase es decorativa.
A3Alert icon box: HTML usa msi sm (20px) pero .an-alert-icon .msi { font-size:14px } lo reduce. Diferencia de 6px.
A4Eyebrow label: HTML usa msi plain (24px) pero .an-fh-eyebrow .msi { font-size:13px } lo lleva a 13px. Diferencia de 11px.
A5Section label: HTML usa msi xs (18px) pero .an-section-lbl .msi { font-size:13px } lo aplana. Mismo patrón que A4.
B1Expiry date: <span class="msi sm" style="font-size:11px">event</span> — el sm declara 20px pero el inline fuerza 11px. Fix: msi xs sin inline.
B2Badge icons (notifications, warning): mismo patrón que B1. Igual que P-05 en portal-screen. Fix: msi xs sin style.
A1–A5: El icono se ve bien porque el CSS corrige el tamaño. El problema es de consistencia: la clase HTML no tiene efecto real. Patrón recomendado: quitar la clase size del HTML y dejar que el CSS de contexto controle el tamaño, o bien eliminar el CSS de contexto y usar siempre la clase.
B1–B2: Bugs reales — dos fuentes de verdad distintas para el mismo valor. El inline gana siempre. Fix inmediato: reemplazar msi sm + font-size:11px por msi xs.
