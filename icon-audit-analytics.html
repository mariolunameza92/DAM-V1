Guía de decisión — cómo resolver cada tipo
⚠ A-type — clase HTML redundante
  • El CSS de contexto ya fija el tamaño: .selector .msi { font-size:Xpx }
  • La clase .sm / .xs en el HTML no tiene efecto visual
  • Opción A (recomendada): quitar la clase del JS — CSS es la única fuente de verdad
  • Opción B: quitar el override CSS y estandarizar usando solo clases HTML
  • Aplicar la misma opción a todos los A-type para consistencia global
🔴 B-type — bug real (inline vs clase)
  • El elemento tiene clase de tamaño (.sm/.xs) y style="font-size:Xpx" diferente
  • El inline siempre gana sobre la clase — la clase nunca tiene efecto
  • Fix inmediato: quitar el style="font-size:..." del JS template literal
  • Si el tamaño inline es el correcto, mover ese valor al CSS del contexto
  • Archivos: analytics.js → _buildSecuritySection(), _buildQualitySection()
